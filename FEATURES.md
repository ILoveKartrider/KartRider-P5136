# P5136 지원 상태

이 문서는 한국 카트라이더 프로토콜 5136 전용 경로의 현재 범위를 정리합니다. “확인됨”은 실제 클라이언트에서 동작을 확인한 기능이고, “구현·자동검증됨”은 패킷 또는 상태 전이 smoke test를 통과했지만 추가 실기기 검증이 필요한 기능입니다.

## 실제 클라이언트에서 확인됨

### 실행과 접속

- P5136 실행 파일과 PIN 기반 빌드 감지
- 접속기의 `KartRider.pin`, `KartRider.xml`, `Profile/kr/launcher.xml` 서버 주소 변경과 PIN 재검증
- 원본 접속 설정의 1회성 `*.pristine.bak` 보존 및 이후 실행의 영구 재패치
- 서버 bind 주소와 클라이언트 광고 주소 분리, LAN 접속과 파생 포트 구성
- 로그인, VIP 초기 응답, 메인 메뉴 진입과 메뉴 간 전환
- 접속기 계정 이름을 실제 게임 닉네임과 프로필 경로에 반영
- 같은 닉네임의 동시 접속 거부

### 로비와 소셜

- 채널별 방 목록 필터링과 방 생성
- 방 참가, 방 설정 변경, 구체적인 맵 선택과 경기 시작
- 마이룸 입장·방문, 방문자 표시·이동 동기화와 마이룸 채팅
- 라이더 찾기와 저장된 라이더 정보 조회

### 카트와 아이템전

- 클라이언트 데이터에서 생성한 로컬 카탈로그를 이용한 전체 인벤토리 제공
- 장착 카트, 캐릭터와 슬롯 수 반영
- 금계 V1, 홍련 V1 등 카탈로그에 기록된 아이템전 카트 능력 반영
- 플랜트 파츠의 P5136 주행 성능값과 게임 모드 조건 반영
- 아이템 상자 획득과 아이템 사용 흐름
- 주행 보고 패킷에서 아이템 획득 시점의 실시간 순위와 위치 해석

## 구현·자동검증됨

### 세션과 멀티플레이 안정성

- TCP 세션 generation과 소유권 검사로 이전 연결의 늦은 disconnect가 새 세션을 지우지 않도록 보호
- generation별 UDP/P2P 첫 바인딩과 오래된 endpoint 갱신 거부
- 채널 generation별 P5136 게임 유형 보존과 통합·무한부스터 채널의 방 목록 분리
- 방 생성·전환 응답의 순서 보장과 중복 응답 억제
- 클라이언트 P2P endpoint 교환 및 서버 fallback relay 경로

### 랜덤 맵

- P5136 클라이언트의 `Data/track_common.rho`에서 트랙 한국어 이름과 랜덤 목록을 읽기 전용으로 로드
- 서버 런처의 `랜덤 맵 목록` 화면에서 코드명 대신 한국어 이름으로 목록 편집
- 목록별 클라이언트 원본값 복원, 전체 선택·해제와 수동 저장
- 스피드전·아이템전 호환 목록 분리와 같은 목록 안에서 한 바퀴 동안 중복 선택 방지
- 수동 설정이 없으면 클라이언트 원본 후보 목록 사용

### 경기와 아이템전

- 경기 결과와 전체 참가자 명단을 먼저 전달한 뒤 시상식 전환을 보내는 P5136 전용 종료 순서
- 시상식 화면에 모든 참가자를 표시하기 위한 결과 snapshot 보존
- 아이템 확률을 클라이언트 원본값에서 로드하고 개인전·팀전 및 실시간 순위 구간별로 편집하는 서버 UI
- P5136 바리케이드 배치 패킷의 길이·종류·소유자·좌표 검증
- 바리케이드 3개 배치 패킷을 아이템 사용자 자신까지 포함해 방 전체에 중계

### 카탈로그와 인벤토리 안전성

- `Profile/KartCatalog.xml` 형식·프로토콜·클라이언트 fingerprint와 완전성 검증
- 카트 이름, 주행 스펙, 슬롯 수, 아이템전 능력, 상점 카탈로그와 플랜트 파츠 성능 로드
- 카트 카탈로그가 없거나 오래됐거나 현재 클라이언트와 다르면 서버 시작 거부
- 캐릭터 목록에서 클라이언트가 모델로 생성할 수 없는 내부·더미 항목을 소유 목록에서 제외해 종료 방지
- 손상되거나 불완전한 새 카탈로그가 이미 로드된 정상 runtime catalog를 덮어쓰지 않도록 보호

### 운영과 진단

- 상세 TCP/UDP 패킷 trace를 UI와 분리된 비동기 파일 writer로 기록
- 서버 UI에는 연결, 상태 전이와 오류 등 요약 로그만 표시
- 서버 시작 시 로그인 TCP, 게임 UDP, P2P UDP와 메신저 TCP를 원자적으로 열고 일부 실패 시 전체 rollback
- 서버 설정과 접속기 설정 분리
- UI에서 핸드셰이크 응답 지연, 물리 프리셋, 속도 등급과 아이템 확률 설정
- S0~S3 및 통합 S7·S8 한국어 표시와 방 이름의 `S0`~`S8` 속도 키워드 지원

## 추가 검증 또는 구현이 필요한 범위

- 여러 명이 장시간 주행할 때의 위치·속도 동기화와 모든 이탈·재접속 순서
- 팀전, AI, 관전자 및 모든 속도·게임 모드 조합
- 새 랜덤 맵 수동 목록, 시상식 전체 명단과 바리케이드 배치의 다양한 실클라이언트·트랙 조합
- 바리케이드 외 모든 설치형·대상 지정형 아이템의 전체 상호작용
- 경기 종료의 모든 순위·팀 점수·보상·중도 이탈 조합
- 친구 추가·목록, 소셜 상태의 장기 보존 및 오프라인 사용자 상호작용
- WAN/NAT 환경에서의 P2P 실패 감지와 완전한 relay 전환
- 패킷 재정렬, jitter buffer 또는 서버 측 위치 보간
- 서버 authoritative 물리 시뮬레이션과 부정행위 방지

## 알려진 제한

- 최초 서버 시작 전에 사용자가 보유한 P5136 데이터에서 `Profile/KartCatalog.xml`을 생성해야 합니다. 자세한 절차는 [README.md](README.md#최초-실행-필수-카트-데이터-xml-추출)를 확인하세요.
- 서버는 P5136 게임 데이터가 있는 폴더에서 빌드와 카탈로그를 검증하므로 독립된 빈 폴더에서는 시작하지 않습니다.
- 공식 릴리스는 게임 클라이언트, PIN, RHO/RHO5, 추출 XML, 패킷 캡처나 분석 산출물을 포함하지 않습니다.
- 실행 파일은 코드 서명되지 않았습니다.
- 전체 패킷 trace는 사용자명, 주소와 인증 데이터를 포함할 수 있습니다. 공개 저장소나 이슈에 올리지 마세요.
- 원격 카트 상태는 클라이언트 주도 P2P와 서버 fallback relay 구조이며 서버 authoritative 물리 시뮬레이션이 아닙니다.
