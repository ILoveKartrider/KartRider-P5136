# KartRider P5136

한국 카트라이더 프로토콜 5136 클라이언트를 위한 테스트용 서버와 Windows 접속기입니다. 로그인, 메인 메뉴, 마이룸, 방 생성·설정, 맵 변경과 경기 시작까지 확인 중인 개발 버전입니다.

이 프로젝트는 [yanygm/Launcher_V2](https://github.com/yanygm/Launcher_V2)의 AFL-3.0 소스를 수정한 파생 프로젝트입니다. 원본 Git 이력을 그대로 게시한 포크가 아니라, 공개 가능한 소스만 새 이력으로 구성했습니다.

## 공개 범위

이 저장소와 공식 릴리스에는 다음 자료를 포함하지 않습니다.

- 원본·언팩·수정 게임 실행 파일과 DLL
- PIN, XML, RHO/RHO5, BML, PK, AES 및 기타 게임 데이터
- 추출한 맵, 카트, 모델, 텍스처, 사운드 등의 리소스
- 패킷 캡처, 실제 packet trace, 실행 로그, 계정·프로필 데이터
- 메모리 덤프, IDA/Ghidra/x64dbg 프로젝트와 주소 맵
- 디컴파일·디스어셈블 출력 및 내부 분석 자료

클라이언트는 제공하지 않습니다. 사용자는 적법하게 보유한 P5136 설치본을 직접 준비해야 합니다.

## 요구 사항

- 빌드: .NET 8 SDK
- 실행: Windows x64와 [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/8.0)
- 지원 대상: 한국 카트라이더 프로토콜 5136 설치본

공식 출력은 framework-dependent입니다. .NET 런타임을 EXE에 포함하지 않아 서버와 접속기 크기를 작게 유지합니다.

## 최초 실행 필수: 카트 데이터 XML 추출

서버를 처음 실행한 뒤 **`서버 시작`보다 먼저 `카트 데이터 XML 추출` 버튼을 한 번 눌러야 합니다.** 서버는 사용자가 보유한 P5136 클라이언트의 `Data/aaa.pk`와 RHO/RHO5를 읽기 전용으로 분석해 `Profile/KartCatalog.xml`을 생성합니다. 이 XML에는 카트 이름·주행 스펙·슬롯 수·아이템전 카트 능력과 상점 아이템 카탈로그가 함께 들어갑니다. 계정별 파일에는 장착 상태만 저장되며, 모든 계정은 이 로컬 카탈로그에서 P5136 패킷으로 안전하게 표현할 수 있는 전체 인벤토리를 동일하게 받습니다. 파일이 없거나 구버전이거나 현재 클라이언트와 맞지 않으면 서버 시작을 거부하므로 버튼으로 다시 추출하세요.

1. 서버 EXE를 P5136 게임 폴더에 둡니다. 해당 폴더 아래에 `Data/aaa.pk`와 원본 RHO/RHO5 파일이 있어야 합니다.
2. 서버 런처를 실행하고 `카트 데이터 XML 추출`을 누릅니다.
3. 로그에 카트 이름·스펙·보유 아이템·능력 해석 수와 `Profile/KartCatalog.xml` 저장 경로가 표시되는지 확인합니다.
4. 추출이 성공한 뒤 네트워크 설정을 확인하고 `서버 시작`을 누릅니다.

P5136 실행 파일에서 검증한 아이템 심볼 ID는 서버의 5136 프로토콜 표에 포함되어 있으므로 `KartRiderU.exe`가 없어도 카트 능력 수치 해석 결과가 동일합니다. 같은 빌드의 `KartRiderU.exe`가 게임 폴더에 있으면 XML 생성 시 상수표를 선택적으로 교차 검증만 합니다. 플랜트 파츠의 주행 성능값도 P5136 클라이언트 `enchantMaterials.xml`에서 검증한 43~46번 전 항목을 서버에 상수표로 포함하며, 게임 모드 조건을 적용합니다. `KartCatalog.xml`, `KartRiderU.exe`, RHO/RHO5와 그 추출물은 사용자 로컬 자료이며 저장소·릴리스·이슈에 올리면 안 됩니다. 클라이언트 데이터가 바뀌었거나 XML fingerprint 오류가 나오면 버튼을 다시 눌러 재생성하세요.

## 빌드

저장소 루트에서 PowerShell로 실행합니다.

```powershell
./scripts/Build.ps1
./scripts/Publish.ps1
```

`Build.ps1`은 공개 금지 자료와 로컬 빌드 정보가 들어오지 않았는지 검사한 뒤 서버·접속기를 빌드하고, KRData codec과 P5136 네트워크 lifecycle smoke test를 실행합니다.

```text
artifacts/publish/server/
artifacts/publish/connector/
artifacts/release/
```

## 사용

서버와 접속기 EXE를 각각 사용할 P5136 `KartRider.exe`가 있는 폴더에 둡니다. 서버는 위의 **최초 XML 추출**을 완료한 다음 시작합니다.

서버 런처는 P236과 같은 방식으로 다음 값을 분리합니다.

- `수신 IPv4 (서버 PC)`: 서버 소켓이 실제로 수신할 로컬 주소
- `광고 IPv4 (이 서버 PC 주소)`: 로그인 후 클라이언트에 전달할 서버 주소
- `로그인 TCP 포트`: 접속기에도 그대로 입력할 실제 로그인 포트

같은 PC에서만 테스트할 때는 bind/advertise 모두 `127.0.0.1`을 사용합니다. LAN에서는 보통 다음과 같이 설정합니다.

```text
수신 IPv4: 0.0.0.0
광고 IPv4: 192.168.1.15   # 서버 PC의 실제 LAN 주소
로그인 TCP:  39312
```

`이 서버 PC LAN IP 자동 입력`은 기본 게이트웨이가 있는 물리 Ethernet/Wi-Fi 어댑터를 우선 선택합니다. Hyper-V, VMware, VirtualBox, Tailscale, WSL 같은 가상 어댑터는 자동 선택에서 제외합니다. 광고 주소에는 클라이언트 PC가 아니라 서버 PC 자신의 주소가 들어가야 합니다.

P5136의 기본 포트 구성은 다음과 같습니다.

| 용도 | 프로토콜 | 기본 포트 |
|---|---|---:|
| 게임 데이터 | UDP | 39311 |
| 로그인 | TCP | 39312 |
| P2P | UDP | 39312 |
| 메신저 | TCP | 39313 |

Windows 방화벽에는 위 TCP/UDP 포트를 각각 허용해야 합니다. 공유기 밖에서 접속하려면 별도의 포트 포워딩이 필요하며, Wi-Fi AP의 client isolation이 켜져 있으면 같은 LAN 장치끼리도 접속할 수 없습니다.

접속기에는 서버 화면의 `광고 IPv4`와 `로그인 TCP 포트`를 그대로 입력합니다. 예전 접속기와 달리 기준 포트 `39311`을 입력하지 않습니다. 새 접속기에서 `39312`는 다시 `+1` 되지 않습니다.

접속기는 게임 실행 전에 파생 메신저 TCP 포트로 서버까지의 LAN 도달 여부를 확인합니다. 접속기 로그에 `서버 TCP 도달 성공`이 없으면 게임 패킷 이전의 IP·포트·방화벽 문제입니다. 이 진단 연결은 로그인 세션을 만들지 않습니다. 도달 확인 후에는 `KartRider.pin`, `KartRider.xml`, `Profile/kr/launcher.xml`의 로그인 대상을 함께 변경하며, PIN은 재암호화 결과를 다시 읽어 검증합니다. 최초 변경 전에 기존 파일은 각각 `*.pristine.bak`으로 한 번만 보존하며 이후 실행에서는 백업을 덮어쓰지 않습니다. 원래 존재하지 않던 파일은 `*.pristine.absent` 표식으로 기록합니다. 변경한 접속 설정은 게임과 접속기가 종료되어도 유지됩니다.

서버는 UI에서 `서버 시작`을 눌렀을 때만 시작합니다. 로그인 TCP, 게임 UDP, P2P UDP, 메신저 TCP가 모두 정상적으로 열려야 상태가 `실행 중`으로 바뀌며, 하나라도 실패하면 앞서 연 리스너도 정리합니다.

## 로컬 설정과 로그

- 서버 런처 설정: `ServerData/server-launcher.json`
- 계정·게임 설정: `Profile/`
- packet trace: 서버 UI에서 지정한 로그 폴더

서버 설정과 접속기 설정을 분리했기 때문에 같은 폴더에서 접속기를 실행해도 실행 중인 서버의 bind/advertise 주소가 바뀌지 않습니다. 위 파일에는 사용자명, 주소 또는 인증 패킷이 기록될 수 있으므로 이슈나 저장소에 첨부하지 마세요.

`ServerData/server-launcher.json`은 PC별 설정이므로 다른 PC에서 복사해 오지 마세요. 이전 버전에서 잘못 저장된 값이 남아 있다면 서버 PC에서 `이 서버 PC LAN IP 자동 입력`을 누른 뒤 광고 주소와 로그인 TCP 포트를 확인하고 다시 저장합니다.
